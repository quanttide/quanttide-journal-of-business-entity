# 2026-03-18 商务拓展标准化日志

## CEO

**商务日志**已经开始在秘书处和商务部之间传递信息。秘书处比较适应这种新的协作模式，商务部还有发消息的惯性。所以“日志驱动协作”的主要难点在于协作习惯的迁移。商务日志的载体是飞书文档；格式是论坛式的盖楼，比飞书话题格式复杂一些，比论坛格式简单一些。接下来的重点是如何智能化地提取有用的信息到**商务档案**里。（用加粗表示正式概念。）

## iFlow

关于信息提取的技术可行性说明：

当前格式特点：
论坛式盖楼格式本质上是有结构的，包含：时间戳、回复层级、参与者信息。这些元数据本身就提供了提取线索。

信息提取的实际路径：
1. 规则提取：先从简单的开始，如提取包含"客户"、"需求"、"决策"等关键词的段落
2. 人工标注：让团队对典型内容进行分类，建立标注数据集
3. 渐进自动化：基于标注数据训练简单的分类模型，逐步提升准确率

协作习惯迁移的技术辅助：
技术上可以通过飞书 API 实现提醒和格式引导，但关键在于降低用户心智负担，而不是增加操作步骤。

建议先做小规模试验，收集真实的日志数据，再决定技术投入的方向和力度。
